1. What is Software Engineering? (2 marks)

Software engineering is a systematic and disciplined approach to the design, development, testing, and maintenance of software systems. It is a branch of engineering focused on creating software that is reliable, efficient, and scalable. Software engineers apply engineering principles, techniques, and methods to ensure that software meets user requirements and quality standards. The goal of software engineering is to produce high-quality software products that meet both functional and non-functional requirements, are easy to maintain, and can be adapted to changing environments over time.

2. Difference Between Program and Software (3 marks)
Program:

A program is a sequence of instructions written in a programming language to perform a specific task. It is a single unit of code that typically solves one problem or completes a specific function, like adding two numbers or printing text.

Example: A program that adds two numbers together.

Software:

Software is a broader term that encompasses a collection of programs, libraries, configuration files, and sometimes documentation. It is not just a single task but a combination of multiple related programs that work together to perform a wide range of functions.

Example: Microsoft Word is software, as it includes the word processor program along with its associated libraries, features, help files, and user manuals.

Key Differences:
  • A program is a single set of instructions or code, while software is a comprehensive set of programs and their components working together.
  • A program solves a specific problem, while software can provide a complete solution to a broader problem domain (like an office suite for various tasks).
  • Software is designed for end users and often includes additional features like a user interface and documentation. A program is often just the logic and code to perform a specific task without those additional elements.
3. Applications of Software (5 marks)

Software plays an integral role in modern society and has a wide range of applications across various industries. Some of the major categories of software applications are:

1. System Software:

System software is designed to manage and operate computer hardware. It provides a platform for running application software and allows users to interact with the hardware.

  • Operating Systems (OS): Examples include Windows, Linux, macOS, and Android.
  • Device Drivers: These allow the operating system to communicate with hardware devices like printers, graphics cards, or network adapters.
  • Utility Software: This includes programs like antivirus software and disk cleanup tools.
2. Application Software:

Application software helps users perform specific tasks, such as productivity work, creativity, or business management.

  • Productivity Software: Examples include Microsoft Office (Word, Excel, PowerPoint) and Google Docs.
  • Multimedia Software: VLC Media Player for video playback, Adobe Photoshop for image editing.
  • Web Browsers: Google Chrome, Mozilla Firefox, and Microsoft Edge.
3. Business Software:

Software that supports business operations, management, and decision-making processes.

  • Enterprise Resource Planning (ERP): Systems like SAP and Oracle.
  • Customer Relationship Management (CRM): Software like Salesforce.
  • Accounting Software: QuickBooks and FreshBooks.
4. Embedded Software:

Embedded software runs on hardware systems with real-time computing requirements.

  • Automobile Systems: Software in cars controls functions like braking and engine performance.
  • Smart Devices: Software runs on smart thermostats, fitness trackers, and appliances.
5. Development Software:

Tools used by developers to create and manage other software applications.

  • Integrated Development Environments (IDEs): Tools like Visual Studio and Eclipse.
  • Version Control Systems: Software like Git and GitHub.
  • Database Management Systems (DBMS): MySQL, PostgreSQL, MongoDB.

1. What is SDLC? (2 marks)

SDLC (Software Development Life Cycle) is a systematic process for planning, creating, testing, deploying, and maintaining software systems. It serves as a framework for ensuring the development process is organized, efficient, and aligned with user needs. The SDLC consists of several stages:

  • Requirement Analysis: Understanding what the user needs.
  • Design: Creating a blueprint for the software.
  • Implementation: Writing and building the actual code.
  • Testing: Verifying that the software works as intended.
  • Deployment: Releasing the software to users.
  • Maintenance: Updating and fixing the software as needed after release.
2. Agile Model in Software Development (8 marks)

The Agile Model is a flexible and iterative approach to software development. It focuses on delivering small, incremental updates to software rather than completing the entire product at once. It emphasizes collaboration, customer feedback, and adaptability.

Key Principles:
  • Software is developed in small cycles called iterations or sprints (2–4 weeks).
  • Continuous involvement of customers ensures software meets their requirements.
  • Quick response to changes, even late in the development process.
  • Regular feedback from stakeholders improves the product.
  • Close collaboration among developers, testers, and customers.
Phases in Agile Development:
  • Concept and Planning: Identify high-level requirements and define user stories.
  • Iteration Execution: Plan, design, code, and test a small portion of the software.
  • Testing and Quality Assurance: Continuous testing ensures high quality.
  • Release: A functional version is released to customers after multiple iterations.
Advantages:
  • Faster Time to Market: Usable software is delivered early in the process.
  • Better Quality: Continuous testing and feedback ensure high-quality outcomes.
  • Adaptability: Responds quickly to changing requirements.
  • Customer Satisfaction: Involves customers at every stage.
Disadvantages:
  • Success depends on skilled team members and effective collaboration.
  • Frequent changes can make long-term planning difficult.
  • Documentation may be limited due to focus on working software.

1. What is SRS? (4 Marks)

SRS (Software Requirements Specification) is a formal document that clearly defines the functional and non-functional requirements of a software system. It acts as a contract between stakeholders and the development team, ensuring that everyone understands what the software must accomplish.

Key Components of SRS:
  • Introduction: Purpose, scope, definitions, and references.
  • System Overview: General description, user characteristics, and constraints.
  • Requirements:
    • Functional Requirements: Describe features and functions.
    • Non-Functional Requirements: Specify performance, security, and usability criteria.
2. Characteristics of a Good SRS (6 Marks)

A good SRS ensures clarity, consistency, and completeness in defining software requirements. Below are the essential characteristics:

  • Completeness: All requirements are included without missing details.
  • Correctness: Aligns with customer needs and expectations.
  • Clarity: Requirements are unambiguous and easily understood.
  • Verifiability: Testable requirements to confirm implementation accuracy.
  • Modifiability: Structured for easy updates without affecting unrelated sections.
  • Consistency: Free of conflicting or contradictory requirements.
  • Traceability: Requirements are traceable to their source and implementation.
  • Feasibility: Realistic and achievable within given constraints.

(5) What is design ? explain different strategies of design [2+8]

1. What is Design? (2 Marks)

Design refers to the process of planning and creating solutions to problems, often resulting in functional, aesthetic, and innovative outcomes. It involves making decisions about structure, layout, functionality, and usability to meet specific goals.

2. Different Strategies of Design (8 Marks)
  • Top-Down Design: This strategy starts with the high-level overview and breaks the system into smaller, more manageable components.

    Example: Creating a website by first planning the overall structure (home page, sections, etc.) and then detailing individual pages.

  • Bottom-Up Design: This approach begins with designing small, detailed components and then integrates them to form the complete system.

    Example: Developing a library of reusable functions or classes before creating the main application.

  • Modular Design: Involves dividing a system into independent modules that can be developed, tested, and maintained separately.

    Example: Building a web application using modular components like header, footer, and sidebars.

  • Iterative Design: Focuses on repeatedly refining the design based on feedback and testing.

    Example: Agile methodology in software development.

  • User-Centered Design (UCD): Prioritizes the needs, preferences, and limitations of the end user throughout the design process.

    Example: Designing an app with intuitive navigation for non-technical users.

  • Responsive Design: Ensures that a system or interface works seamlessly across various devices and screen sizes.

    Example: Designing a website that adapts to desktop, tablet, and mobile screens.

  • Aesthetic Design: Focuses on the visual appeal and emotional impact of the design.

    Example: Creating a modern, visually attractive website with appropriate colors, fonts, and layouts.

  • Sustainable Design: Aims to minimize environmental impact and promote sustainability through efficient use of resources.

    Example: Choosing energy-efficient hardware or eco-friendly materials in physical product design.

1. What is Software Quality? (2 Marks)

Software quality refers to the degree to which a software product meets the specified requirements, user expectations, and performance standards. It ensures the software is reliable, efficient, maintainable, and user-friendly.

Key Attributes of Software Quality:

  • Functionality: Meets the intended purpose and requirements.
  • Reliability: Performs consistently under specified conditions.
  • Usability: Is easy to use and understand.
  • Maintainability: Can be updated and fixed efficiently.
2. Capability Maturity Model (CMM) (8 Marks)

The Capability Maturity Model (CMM) is a framework used to assess and improve software development processes. It helps organizations enhance their ability to deliver high-quality software by providing a structured path for process improvement.

Key Features of CMM:

  • Helps organizations improve process consistency and predictability.
  • Provides a roadmap for process improvement.
  • Focuses on maturity levels to evaluate the organization's capabilities.

Maturity Levels of CMM:

  • Level 1: Initial: Processes are ad-hoc and unpredictable. Success depends on individual efforts.
  • Level 2: Repeatable: Basic project management processes are established. Success can be repeated for similar projects.
  • Level 3: Defined: Processes are documented and standardized across the organization.
  • Level 4: Managed: Processes are measured and controlled using quantitative data.
  • Level 5: Optimizing: Continuous process improvement is implemented based on feedback and innovation.

What is Software Testing? (2 marks)

Software testing is the process of evaluating a software application to ensure that it functions as expected, meets the requirements, and is free of defects or bugs. It involves running the software in various conditions, checking its behavior, and validating that the outcomes are accurate and consistent. Testing helps identify issues early, reduce risks, and ensure the quality of the final product.

Different Levels of Testing (8 marks)

There are several levels of software testing, each focusing on specific aspects of the software to ensure its quality and reliability:

1. Unit Testing:

Unit testing focuses on individual components or units of the software, such as functions or methods. The goal is to ensure that each unit performs as expected in isolation.

  • Example: Testing a function that adds two numbers to ensure it returns the correct sum.
  • Performed By: Developers
2. Integration Testing:

Integration testing checks if different modules or components of the software work together as expected, ensuring that data flows correctly between units and they interact properly.

  • Example: Testing the interaction between a database and a login form to verify that user credentials are properly checked.
  • Performed By: Developers or QA Engineers
3. System Testing:

System testing tests the entire software system as a whole, ensuring that all integrated components function correctly and meet specified requirements.

  • Example: Testing a web application to ensure all features (e.g., user registration, payment processing) work together as expected.
  • Performed By: QA Engineers
4. Acceptance Testing:

Acceptance testing is performed to determine whether the software meets the business requirements and is ready for deployment. It is typically done by end users or stakeholders.

  • Example: A client testing the features of a project management tool before release.
  • Performed By: End-users or Business Analysts
5. Regression Testing:

Regression testing ensures that new code changes or additions do not affect the existing functionalities of the software.

  • Example: After adding a new feature to an app, running tests to ensure previous functionalities still work correctly.
  • Performed By: QA Engineers
6. Alpha Testing:

Alpha testing is an internal testing phase performed by the development team to identify major bugs before releasing the software to external testers.

  • Example: Internal testing by the development company before releasing the software to beta testers.
  • Performed By: Developers and QA Engineers
7. Beta Testing:

Beta testing involves releasing the software to a select group of external users to test in real-world environments and gather feedback before final release.

  • Example: A social media app tested by a group of users for feedback on usability.
  • Performed By: External Users (beta testers)

Software Management Process (5 Marks)

The software management process refers to the systematic approach to managing the development and maintenance of software throughout its lifecycle. It involves planning, design, development, testing, deployment, and maintenance, ensuring that software projects are completed on time, within budget, and meet required quality standards.

Key Components:
  • Project Planning: Establishing scope, objectives, and resources.
  • Requirement Analysis: Gathering and defining requirements.
  • Software Design: Creating architecture and selecting technologies.
  • Development: Coding and implementing the software.
  • Testing: Ensuring software meets requirements and is bug-free.
  • Deployment: Releasing the software for production.
  • Maintenance: Regular updates and fixing bugs.
Software Re-engineering (5 Marks)

Software re-engineering is the process of modifying an existing software system to improve performance, maintainability, or adaptability without changing its functionality. The goal is to extend the software’s lifecycle and make it more cost-effective and suitable for modern needs.

Re-engineering Process:
  • Reverse Engineering: Analyzing the existing system to understand its operation.
  • Restructuring: Improving the software's structure for better maintainability.
  • Forward Engineering: Updating to modern standards and technologies.
  • Re-documentation: Updating documentation to reflect changes.
  • Testing: Ensuring no defects are introduced after re-engineering.

(b)Cohesion and Coupling (5 Marks)

Cohesion

Cohesion refers to the degree to which the elements within a module or component are related to each other. A highly cohesive module has components that are well-related and focused on a single task or responsibility.

Types of Cohesion:
  • High Cohesion: A module performs a specific, well-defined task.
  • Low Cohesion: A module performs unrelated tasks, leading to complexity and difficulty in maintenance.
Example:

A class that manages user authentication should have high cohesion, focusing on tasks like login and password validation, without involving database management or UI rendering.

Benefits of High Cohesion:
  • Improved maintainability and readability.
  • Better reusability of focused modules.
  • Reduced complexity and fewer side effects when changes are made.
Coupling

Coupling refers to the degree of dependence between two modules. A system with low coupling means that modules have minimal dependencies, whereas high coupling means that modules are tightly dependent on each other.

Types of Coupling:
  • Low Coupling: Modules have minimal dependencies on each other.
  • High Coupling: Modules are heavily dependent on each other.
Example:

If two classes (Class A and Class B) interact via well-defined interfaces, they exhibit low coupling. Direct modification of each other’s internals signifies high coupling.

Benefits of Low Coupling:
  • Easier modification and extension of the system.
  • Better modularity and independence between components.
  • Facilitates testing as modules can be tested in isolation.

(c)Black Box and White Box Testing (5 Marks)

Black Box Testing

Black Box Testing, also known as functional or behavioral testing, is a method where the tester evaluates the functionality of the software without knowing its internal workings. The focus is on what the software does rather than how it does it.

Key Features:
  • The tester does not need knowledge of the code or architecture.
  • Tests are based on requirements and specifications.
  • Involves testing inputs and outputs.
  • Focuses on the external behavior of the system.
Types of Black Box Testing:
  • Functional Testing: Ensures that the software meets the specified requirements.
  • Regression Testing: Checks that new changes have not introduced defects.
  • System Testing: Tests the entire system's compliance with functional and non-functional requirements.
Advantages:
  • The tester can focus purely on the user experience.
  • Ideal for acceptance testing where the user’s perspective is critical.
  • Doesn’t require knowledge of the internal structure of the software.
Example:

Testing a login form where the tester enters a valid username and password and checks if the system allows access. The tester does not need to know the implementation details of the authentication process.

White Box Testing

White Box Testing, also known as structural or glass box testing, is a testing method where the tester has knowledge of the internal structure and code of the software. The focus is on the internal workings of the software and ensures that the code functions as expected.

Key Features:
  • The tester needs knowledge of the code, architecture, and logic.
  • Involves testing internal functions, pathways, and logic.
  • Aims to identify vulnerabilities, bugs, and code inefficiencies.
Types of White Box Testing:
  • Unit Testing: Testing individual units or components of the code.
  • Integration Testing: Ensuring different modules work together as expected.
  • Code Coverage Testing: Ensures that all code paths and branches are tested.
Advantages:
  • Helps identify hidden errors in the code early.
  • Can lead to better performance and code optimization.
  • Can test the internal workings and logic directly.
Example:

Testing a function that calculates the sum of an array. The tester would check if all parts of the function, including loops and conditionals, are covered in the test cases.

Black Box vs White Box Testing

Black Box Testing focuses on testing the software from the end user's perspective, ensuring that it behaves as expected under various input conditions, while White Box Testing focuses on testing the internal logic and structure of the code.

Black Box Testing does not require knowledge of the code, whereas White Box Testing requires an understanding of the code and its internal workings.

(d)4P Model [5]

Product

The product refers to the goods or services offered by a business to meet the needs or desires of its target market.

Key Considerations:
  • The quality, features, design, brand, and warranty.
  • Product differentiation based on customer needs.
  • Packaging and customer service.
Example:

Apple’s iPhone is a premium product with high quality, innovative features, and a unique design, making it desirable in the smartphone market.

Price

Price refers to the amount of money a customer has to pay to acquire the product. It plays a crucial role in determining the perceived value of the product and affects demand.

Key Considerations:
  • Pricing strategies such as penetration pricing, skimming, or value-based pricing.
  • Discounts, offers, and payment plans.
  • Price elasticity (sensitivity to price changes).
Example:

A luxury brand like Rolex uses premium pricing to convey exclusivity and high value, while budget brands may use competitive pricing to appeal to cost-conscious consumers.

Place

Place refers to how and where the product is distributed and made available to customers. This element ensures that the product reaches the right customers at the right time and in the right location.

Key Considerations:
  • Distribution channels such as retail stores, e-commerce platforms, and wholesalers.
  • Location decisions for physical stores or online presence.
  • Logistics and supply chain management.
Example:

Amazon uses an online platform for distribution, allowing customers worldwide to purchase products with ease. In contrast, traditional retailers like Walmart rely on physical stores for distribution.

Promotion

Promotion involves all activities and strategies used to raise awareness and persuade customers to purchase the product. It includes advertising, public relations, sales promotions, and direct marketing.

Key Considerations:
  • The channels and messages used to communicate with the target audience.
  • Promotional strategies like discounts, contests, loyalty programs, and advertisements.
  • Seasonal campaigns or special product launches.
Example:

Coca-Cola often runs promotional campaigns that feature seasonal ads, sponsorships, and social media engagement to enhance brand visibility and customer engagement.

Conclusion

The 4P Model provides a comprehensive approach to developing an effective marketing strategy by balancing product, price, place, and promotion. By addressing these elements, companies can optimize their market offerings and create a competitive advantage.
